Multicolumn

Multicolumn is a CSS specification that's been around for a long time. You can do a layout like this with it. Sadly, most people aren't using it, for several reasons: 1) Most designers and front-end developers don't know this is possible; 2) the implementations are very buggy and have real interop problems, and 3) Multicolumn can likely only be used for very short amounts of text, for usability reasons. I'm working on solving problem 1 and 2. But I wonder about problem 3. Does this layout work? Will readers be ok with scrolling down the page to read the first column, then scrolling back up to the top to get to the top of the second column, scrolling back down the page, scrolling back up to the top again, etc???? I'm not sure that they will want to do this. Instead I think we need a way to be able to do a multicolumn layout without all that scrolling. I think we need the ability to do what CSS Regions allows See the next example to see what I mean.
